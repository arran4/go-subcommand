# gosubc Agent Skill

This skill teaches AI agents how to correctly use the `gosubc` CLI tool.

## What is gosubc?

`gosubc` is a code generation tool for creating dependency-less subcommands in Go applications. It works by parsing specifically formatted comments on Go functions to generate a CLI structure.

## Core Operations

### 1. Generating Code
The most common operation is generating the subcommand code:
```bash
gosubc generate
```
Agents should typically run this in the root of the Go module (where `go.mod` is located) or pass `--dir` to specify the directory. If a `generate.go` file exists with `//go:generate gosubc generate`, simply running `go generate` is preferred.

### 2. Validating Code
Before submitting code, agents should validate the subcommand definitions to ensure comments are parsed correctly without errors:
```bash
gosubc validate
```
This is a safe, read-only operation to check for parsing errors.

### 3. Listing Subcommands
To verify the parsed subcommand structure:
```bash
gosubc list
```
This lists all detected subcommands.

### 4. Formatting Source Comments
To format subcommand source comments to match standard formatting style:
```bash
gosubc format-source-comments
```
Use this to fix comment alignment and formatting issues.

## Traps and Common Mistakes for Agents

1. **gosubc is not a dependency:** NEVER add `github.com/arran4/go-subcommand` as a dependency in `go.mod` for applications using `gosubc` to build their CLI. `gosubc` is a build-time tool. The generated code is self-contained.
2. **Comment placement:** The `// FuncName is a subcommand ...` directive must be immediately preceding the function, without blank lines, or the function will not be parsed.
3. **Running the generator:** `gosubc` must be executed to regenerate files whenever comments defining commands or parameters are modified.
4. **Bootstrapping / Self-hosting:** When modifying `gosubc` itself, you must run `go run ./cmd/gosubc generate` to regenerate its own CLI files.
5. **DO NOT edit generated files:** Files with the header `// Code generated by github.com/arran4/go-subcommand/cmd/gosubc. DO NOT EDIT.` should NEVER be edited directly. Edit the source code comments and run `gosubc generate` instead.
